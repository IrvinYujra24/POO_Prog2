Introducción a la Programación Orientada a Objetos

¿Qué es la POO?

La Programación Orientada a Objetos (POO) es un paradigma de programación que utiliza "objetos" para diseñar aplicaciones y programas de software. Los objetos son instancias de clases, que pueden contener datos en forma de campos (también llamados atributos o propiedades) y código en forma de procedimientos (métodos).

Concepto clave: La POO se basa en cuatro principios fundamentales: Encapsulamiento, Abstracción, Herencia y Polimorfismo. Estos principios ayudan a crear código más modular, reutilizable y fácil de mantener.

Conceptos Básicos

Clase: Una clase es una plantilla o modelo para crear objetos. Define las propiedades y comportamientos que los objetos de esa clase tendrán.

Objeto: Un objeto es una instancia de una clase. Es una entidad con estado (atributos) y comportamiento (métodos).

Atributo: Las características o propiedades de un objeto.

Método: Las acciones o comportamientos que un objeto puede realizar.

Ejemplo Práctico

Veamos un ejemplo básico de una clase en Java:

public class Coche {
    // Atributos
    private String marca;
    private String modelo;
    private int año;
    private boolean encendido = false;

    // Constructor
    public Coche(String marca, String modelo, int año) {
        this.marca = marca;
        this.modelo = modelo;
        this.año = año;
    }

    // Métodos
    public void encender() {
        encendido = true;
        System.out.println("El coche se ha encendido");
    }

    public void acelerar() {
        if (encendido) {
            System.out.println("El coche está acelerando");
        } else {
            System.out.println("Primero debes encender el coche");
        }
    }

    // Getters y setters
    public String getMarca() {
        return marca;
    }
}

La POO organiza el software en objetos que interactúan entre sí, como piezas de un rompecabezas

Encapsulamiento

¿Qué es el Encapsulamiento?

El encapsulamiento es un principio de la POO que consiste en ocultar los detalles internos de un objeto y exponer solo lo necesario a través de una interfaz pública. Es como una cápsula que protege los datos internos del objeto.

Beneficios del encapsulamiento: Mejora la seguridad, facilita el mantenimiento, permite cambiar la implementación interna sin afectar al código que usa la clase, y previene que los datos se modifiquen de forma inapropiada.

Cómo Implementar el Encapsulamiento

En Java, el encapsulamiento se logra utilizando modificadores de acceso:

  • private: Accesible solo dentro de la misma clase
  • protected: Accesible dentro de la misma clase y sus subclases
  • default (package-private): Accesible dentro del mismo paquete
  • public: Accesible desde cualquier lugar

Ejemplo Práctico

Veamos cómo aplicar el encapsulamiento en una clase de cuenta bancaria:

public class CuentaBancaria {
    // Atributos privados - encapsulados
    private String numeroCuenta;
    private double saldo;
    private String titular;

    public CuentaBancaria(String numeroCuenta, String titular) {
        this.numeroCuenta = numeroCuenta;
        this.titular = titular;
        this.saldo = 0;
    }

    // Métodos públicos para interactuar con la cuenta
    public void depositar(double cantidad) {
        if (cantidad > 0) {
            saldo += cantidad;
            System.out.println("Depósito de $" + cantidad + " realizado");
        } else {
            System.out.println("La cantidad debe ser positiva");
        }
    }

    public boolean retirar(double cantidad) {
        if (cantidad > 0 && cantidad <= saldo) {
            saldo -= cantidad;
            System.out.println("Retiro de $" + cantidad + " realizado");
            return true;
        } else {
            System.out.println("Fondos insuficientes o cantidad inválida");
            return false;
        }
    }

    // Getters (accesores)
    public double getSaldo() {
        return saldo;
    }

    public String getNumeroCuenta() {
        return numeroCuenta;
    }

    public String getTitular() {
        return titular;
    }
}

El encapsulamiento es como una caja fuerte: protege los datos valiosos y solo permite acceso controlado

Herencia

¿Qué es la Herencia?

La herencia es un mecanismo que permite crear nuevas clases basadas en clases existentes. La clase nueva (subclase o clase derivada) hereda los atributos y métodos de la clase existente (superclase o clase base) y puede agregar nuevos atributos y métodos o modificar los existentes.

Principio de reutilización: La herencia promueve la reutilización de código, permitiendo que las subclases aprovechen la funcionalidad existente sin tener que volver a escribirla. Esto sigue el principio DRY (Don't Repeat Yourself).

Jerarquía de Clases

La herencia crea una relación "es un" entre clases. Por ejemplo, un Círculo es una Forma, un Perro es un Animal.

Ejemplo Práctico

Implementemos una jerarquía de clases para formas geométricas:

public abstract class Forma {
    protected String color;

    public Forma(String color) {
        this.color = color;
    }

    public abstract double calcularArea();

    public abstract double calcularPerimetro();

    public String getColor() {
        return color;
    }
}

public class Circulo extends Forma {
    private double radio;

    public Circulo(String color, double radio) {
        super(color);
        this.radio = radio;
    }

    @Override
    public double calcularArea() {
        return Math.PI * radio * radio;
    }

    @Override
    public double calcularPerimetro() {
        return 2 * Math.PI * radio;
    }
}

public class Rectangulo extends Forma {
    private double ancho, alto;

    public Rectangulo(String color, double ancho, double alto) {
        super(color);
        this.ancho = ancho;
        this.alto = alto;
    }

    @Override
    public double calcularArea() {
        return ancho * alto;
    }

    @Override
    public double calcularPerimetro() {
        return 2 * (ancho + alto);
    }
}

La herencia es como una familia: las generaciones más jóvenes heredan características de sus ancestros

Polimorfismo

¿Qué es el Polimorfismo?

El polimorfismo es la capacidad de un objeto para tomar muchas formas. En POO, permite que objetos de diferentes tipos respondan al mismo mensaje de maneras diferentes. La palabra proviene del griego y significa "muchas formas".

Principio clave: El polimorfismo permite escribir código más flexible y reutilizable. Un mismo método puede comportarse de manera diferente dependiendo del tipo de objeto que lo invoca.

Tipos de Polimorfismo

Existen dos tipos principales de polimorfismo:

En tiempo de compilación

Sobrecarga de métodos: Múltiples métodos con el mismo nombre pero diferentes parámetros. El compilador decide cuál método llamar.

En tiempo de ejecución

Anulación de métodos: Una subclase proporciona una implementación específica de un método ya definido en su superclase.

Ejemplo Práctico

Demostración del polimorfismo con formas geométricas:

public class DemoPolimorfismo {
    public static void main(String[] args) {
        // Array de referencias a Forma
        Forma[] formas = new Forma[3];

        formas[0] = new Circulo("Rojo", 5.0);
        formas[1] = new Rectangulo("Azul", 4.0, 6.0);
        formas[2] = new Triangulo("Verde", 3.0, 8.0);

        // Polimorfismo en acción
        for (Forma forma : formas) {
            System.out.println("Color: " + forma.getColor());
            System.out.println("Área: " + forma.calcularArea());
            System.out.println("Perímetro: " + forma.calcularPerimetro());
            System.out.println("--------");
        }
    }
}

En este ejemplo, aunque todas las referencias son de tipo Forma, el método calcularArea() que se ejecuta depende del tipo real del objeto en tiempo de ejecución.

El polimorfismo es como un rompecabezas: muchas piezas diferentes que encajan en el mismo espacio

Ejercicios de Nivel Medio

Sistema de Empleados
Medio

Crea una jerarquía de clases para un sistema de empleados. Define una clase base Empleado con atributos nombre, salario y métodos para calcular salario. Luego crea clases derivadas EmpleadoTiempoCompleto y EmpleadoPorHoras que implementen el cálculo de salario de manera diferente.

public abstract class Empleado {
    protected String nombre;
    protected double salarioBase;

    public Empleado(String nombre, double salarioBase) {
        this.nombre = nombre;
        this.salarioBase = salarioBase;
    }

    public abstract double calcularSalario();
}

public class EmpleadoTiempoCompleto extends Empleado {
    public EmpleadoTiempoCompleto(String nombre, double salarioBase) {
        super(nombre, salarioBase);
    }

    @Override
    public double calcularSalario() {
        return salarioBase;
    }
}

public class EmpleadoPorHoras extends Empleado {
    private int horasTrabajadas;
    private double tarifaPorHora;

    public EmpleadoPorHoras(String nombre, double tarifaPorHora) {
        super(nombre, 0);
        this.tarifaPorHora = tarifaPorHora;
    }

    public void setHorasTrabajadas(int horas) {
        horasTrabajadas = horas;
    }

    @Override
    public double calcularSalario() {
        return horasTrabajadas * tarifaPorHora;
    }
}

Sistema de Gestión de Biblioteca
Avanzado

Desarrolla un sistema completo de gestión de biblioteca que incluya clases para Libros, Revistas, Usuarios, Préstamos, etc. Utiliza todos los principios de POO: encapsulamiento, herencia, polimorfismo y abstracción. Implementa un sistema de préstamos con diferentes tipos de usuarios (estudiantes, profesores) que tengan diferentes políticas de préstamo.

public abstract class Publicacion {
    protected String titulo;
    protected String autor;
    protected boolean disponible;

    public abstract int getDiasPrestamo();
}

public class Libro extends Publicacion {
    private String isbn;
    private int paginas;

    @Override
    public int getDiasPrestamo() {
        return 14; // 2 semanas
    }
}

public abstract class Usuario {
    protected String nombre;
    protected String id;
    protected List prestamos;

    public abstract boolean puedePrestarMas();
}

public class Estudiante extends Usuario {
    @Override
    public boolean puedePrestarMas() {
        return prestamos.size() < 5;
    }
}

public class Profesor extends Usuario {
    @Override
    public boolean puedePrestarMas() {
        return prestamos.size() < 10;
    }
}

public class Biblioteca {
    private List publicaciones;
    private List usuarios;

    public boolean prestarPublicacion(Usuario usuario, Publicacion publicacion) {
        if (publicacion.disponible && usuario.puedePrestarMas()) {
            publicacion.disponible = false;
            usuario.prestamos.add(publicacion);
            return true;
        } else {
            return false;
        }
    }
}
Juego de Rol (RPG)
Avanzado

Desarrolla un sistema de juego de rol que incluya clases para Personajes, Enemigos, Armas, Hechizos, etc. Implementa un sistema de combate utilizando polimorfismo, donde diferentes tipos de personajes y enemigos tengan diferentes comportamientos de ataque. Utiliza patrones de diseño como Strategy para diferentes tipos de ataques.

public interface EstrategiaAtaque {
    void atacar(Personaje atacante, Personaje objetivo);
}

public class AtaqueFisico implements EstrategiaAtaque {
    @Override
    public void atacar(Personaje atacante, Personaje objetivo) {
        int dano = atacante.fuerza - objetivo.defensa;
        if (dano < 1) dano = 1;
        objetivo.vida -= dano;
        System.out.println(atacante.nombre + " inflige " + dano + " puntos de daño");
    }
}

public abstract class Personaje {
    protected String nombre;
    protected int vida, fuerza, defensa;
    protected EstrategiaAtaque estrategiaAtaque;

    public abstract void atacar(Personaje objetivo);
}

public class Guerrero extends Personaje {
    public Guerrero(String nombre) {
        this.nombre = nombre;
        this.estrategiaAtaque = new AtaqueFisico();
    }

    @Override
    public void atacar(Personaje objetivo) {
        estrategiaAtaque.atacar(this, objetivo);
    }
}
